### AlarmRuleView.vue 设计

#### 1. 列表页
- **工具栏**：搜索框 + 新建告警规则按钮
- **表格列**：规则名称 / 部署范围(N个摄像头) / 生效时段 / 通知方式 / 24h触发次数 / 状态(启用/停用) / 操作(编辑/启用停用/删除)
- 行操作：编辑、启用/停用切换、删除

#### 2. 新建弹窗（5 步骤向导）
使用 `a-modal` + 自定义步骤条，宽度 720px。

**Step 1 — 基础信息**
- 规则名称（输入框）
- 告警等级（单选：紧急/警告/提示）
- 描述（文本域）

**Step 2 — 场景和算法**
- 左侧场景列表（卡片按钮）：入侵检测 / 烟火检测 / 人员行为 / 车辆管理 / 物品安全
- 右侧该场景下的算法（多选 checkbox 卡片）
- 可跨场景多选算法

**Step 3 — 生效摄像头**
- 左侧：区域树（`a-tree`，复用 video.mock.ts 的 areaTree 结构）
- 右侧：选中区域下的摄像头卡片网格，可多选（checkbox）
- 顶部显示已选数量

**Step 4 — 生效时段**
- 单选切换：全天 / 工作日 / 夜间 / 自定义
- 自定义：简化为一个 7×24 的时段表格（行=周一至周日，列=0-23时），点击格子切换选中
  - 简化交互：点击拖选区间，或按天「全天/清空」快捷按钮

**Step 5 — 通知对象**
- 通知方式（多选 checkbox）：站内信 / 短信 / 邮箱 / 钉钉 / 企业微信 / Webhook
- 通知人员（多选标签输入）
- 去重开关（默认关闭）：开启后显示「同一摄像头 N 分钟内只推送一条」输入框

#### 3. 步骤导航
- 底部：上一步 / 下一步 / 保存启用
- 步骤指示器在顶部

### Mock 数据（rule-mock.ts）
- `AlarmRule` 接口 + 6 条规则数据
- `scenes` 场景 + 算法映射
- `cameraTree` 摄像头树数据（复用 video 区域结构）
- `notifyMethods` 通知方式选项

### 设计规范
- 弹窗 720px 宽，圆角 12px
- 步骤条：紫色主题
- 表格：白底，表头 #FAFBFC，行 hover #FAF9FF
- 按钮遵循 design.html（主按钮 32px/radius 6px）
- 时段网格：紧凑格状，选中紫色，未选中浅灰
